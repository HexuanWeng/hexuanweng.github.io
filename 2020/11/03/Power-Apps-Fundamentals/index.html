Introduction to Power Platform

In this module, you will:

  • Learn the components and features of Power Platform
  • Identify when to use each Power Platform component application to create business solutions
  • Learn the value of using the Power Platform to create business solutions

What is Power Platform?

Act by building custom apps in Power Apps, automate processes based on the data you collected in Power Automate, and analyze the data you have collected in Power BI.
The power platform can add value to any business by helping you to analyze, act and automate.

  1. Power Apps
    • build custom apps for business needs
    • web and mobile applications
  2. Power Automate (used to named as Microsoft Flow)
    • automated workflows between applications and services
      • communication
      • data collections
      • decision approvals
  3. Power BI
    • business analytics service
    • insights leg of the Power Platform
    • can potentially replace a standing meeting to report out on company metrics:
      • sales data
      • progress against goals
      • employee performance
  4. Power Virtual Agents
    • create powerful chatbots
    • users can enable chatbots to perform an action by simply calling a Power Automate flow

Features

  1. AI Builder
    • workflows
    • Power apps
  2. Common Data Service
    • let users securely store and manage data from multiple sources and integrate that data
    • foundation that enables the consolidation, display and manipulation of data
    • A type of Tabular Data
  3. Connectors
    • enable to connect apps, data and devices in the cloud
    • bridge from data source to app, workflow and dashboard
    • understand the types of data sources they connect
      • Tabular Data (in a structured table foramt)
        • Common Data Service
        • SharePoint
        • SQL Server
      • Function-based data (a table of data but offer more extensive action such as the ability to send an email, update permissions or create a calendar event)
        • Office 365 Users
        • Project Online
        • Azure Blob Storage
    • types of connectors
      • Standard
        • SharePoint
        • Outlook
        • YouTube
      • Premium (require additional licensing for your app and/or users)
        • SQL Server
        • Survey Money
        • Mail Chimp
    • users can also use APpSource to cource and install apps and use the connectors to non-Microsoft services
    • Connectors can provide input and output between the data source and the Power Platform
    • once estabilished a data source and configured concector, two types of operations:
      • Triggers
        • only used in Power Automate
        • can be time based
        • prompt a flow to begin
      • Actions
        • used in Power Automate and Power Apps
        • be prompted by the user or a trigger and allow interaction with your data source by some function
    • Custom Connectors
      • call a public available API
      • advantage: used in different platforms
        • Power Apps
        • Power Automate
        • Azure Logic Apps

Data loss prevention, compliance, privacy and accessibility

  1. data loss prevention policies
  2. connectors type in the context of the organization
    • Business
    • Non-Business (defaulted set)
    • Blocked
  3. Data protection
    • all public endpoints are secured using industry-standard TLS
    • the Power Platform does fully support the GDPR (The European Union General Data Protection Regulation)
  4. Accessibility in the Power Platform
    • an accessible canvas app will allow users with vision, hearing and other impairments to successfully use the app

Summary and Key take aways

  1. Power Platform is a system that enables users to do three key actions on data that help them drive business:
    • Analyze: gain insights from data
    • Act: drive intelligent business processes via apps they build
    • Automate: automate busness processes
  2. Power BI hleps you analyze and visualize data on a unified platform with data from internal and external sources
  3. Power Apps helps you build and deploy customized apps that work across web and mobile, embedded or standalone, on any device
  4. Connectors are brighes that allow you to send information from your data source to your app or workflow and back
  5. Power Automate helps you create automation workflows, from simple to advanced scenarios
